# 管理员审计中心 v1.1 实施计划

## 目标

把压测页从“列表内展开原始 JSON”升级为可读的运行详情：列表保持轻量，点击后通过详情 API 读取并展示 L1-L4、资源、清理和停止原因；同时补齐详情加载、失败、空数据和窄屏布局。后端读取数据库中的 `safe_summary` 时必须再次经过严格 schema 校验，历史或漂移脏数据只能安全降级，不能泄漏额外字段。

## 任务边界

- 前端：压测记录列表、结构化详情视图、安全类型、详情交互与状态。
- 后端：压测详情/列表读取时的严格安全投影与兼容降级。
- 不扩展压测执行能力，不改普通聊天页，不增加写操作或导出。

## 用例矩阵

| case | 场景 | 自动化断言 | 生产验收 |
|---|---|---|---|
| PERF-DETAIL-01 | schema v2 完整运行 | L1-L4、resources、cleanup、时间与状态可读展示 | 最新 completed 记录可打开 |
| PERF-DETAIL-02 | 详情加载/失败 | 加载状态、重试按钮、403 清空管理页 | 网络请求成功，无 console error |
| PERF-DETAIL-03 | stopped/空阶段 | stop reasons 可见，空分组不渲染伪数据 | 历史记录可安全展示 |
| PERF-DETAIL-04 | 历史或非法安全汇总 | 后端严格校验并固定码降级，不返回额外字段 | 不在生产制造脏数据，仅自动化验证 |
| PERF-DETAIL-05 | 窄屏 | 卡片单列、长 run/model id 换行、无横向页面溢出 | 复用生产标签临时窄屏检查 |
| PERF-DETAIL-06 | 刷新恢复 | 刷新后列表和详情仍可读取 | 生产刷新后再次打开同一记录 |

## 验收门槛

- 前后端目标测试先红后绿，相关全量测试通过。
- 前端 ESLint 与生产 build 通过；后端 Ruff 与架构检查通过。
- 两个子仓按需提交、push，CI/CD 和部署 smoke 成功。
- 真实登录态 Chrome 只复用现有 Fusion 管理页，记录 URL、预期、实际、console 和刷新结果。
